Protocol

A run you can execute end to end.

  1. 1. Prove

    Build the Lean projects and run TLC on the TLA+ specs. A proof that fails to check is a failed claim, full stop.

  2. 2. Run

    Execute each repo's test suite, weighting the tamper, replay, and fuzz cases. Regenerate the metrics and diff the JSON.

  3. 3. Break

    Construct an input the gate should reject. Confirm it is rejected. Then read the stated limitations and test whether any are understated.

Weighting

How to score what you find.

Signals are not equal. Weight by how hard each is to fake, not by how large it looks.

SignalWeight
A machine-checked proof that compiles.Highest. It cannot be faked.
An adversarial test that reproduces; a certificate that matches on replay.High.
Line counts, commit counts, stars, follower counts.Near zero. The candidate does not weight these, and neither should you.
Any claim with no attached command.Discount it, and flag it in your report.

Adversarial

Try to break it. That is the point.

A claim you cannot break is worth more than one nobody tested. I would rather you surface a real weakness than miss it, so here is where to push.

Specification gaps

A proof is only as strong as what it states. Check whether a theorem proves a simplified property while the live system relies on a richer one.

Boundary coverage

Look for an interface that reaches an effect without passing the intended gate or receipt check.

Replay binding

Confirm a replay certificate is actually bound to the critical path, not to a convenient summary of it.

Estimate honesty

The value figures are replacement-cost estimates with a stated method. Check that the caveats match the numbers.
